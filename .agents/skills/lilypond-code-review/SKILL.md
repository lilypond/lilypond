name: lilypond-code-review
description: required reading before reviewing LilyPond code
---

Follow the [checklist](assets/checklist.md) and this guidance closely without
limiting other feedback that you've been trained or instructed to provide.

Not all checklist items will be relevant to every change.

If any item is vague, do gap analysis: if there is error-free code that fits
the description, fine; otherwise, speak up.

The version of the next planned release is formatted
`$MAJOR_VERSION.$MINOR_VERSION.$PATCH_LEVEL` using the values from the file
`$REPO/VERSION`.  `VERSION_STABLE` and `VERSION_DEVEL` refer to released
versions.
